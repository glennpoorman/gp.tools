Gp.Basic.checkForHTML5()
Returns true if the browser appears to support HTML5 and false if it doesn't.
Gp.Basic.html5Message()
Checks to see if the browser supports HTML5 and if it doesn't, looks for an element in the page with the id "gp-upgrade" and displays a message recommending that the user upgrade their browser to the 21st century.

Note that this function is automatically called on document load meaning there will never be a reason to call it directly. To take advantage, you only need to put an element (such as a DIV or SPAN) in your document with the id "gp-upgrade" as follows:
<span id="gp-upgrade"></span>
With that element in place, the rest is automatic. Honestly ... at this point none of these websites are money making ventures for me so I have no interest in supporting someone who won't accept a free browser upgrade in order to get an up to date web experience. I mean really!
Gp.Basic.emailLink(id, user, host, link)
Inserts an email into your HTML in such a way that makes it difficult for spam bots to find.
id - id of the element to add the email link to.
user - user name portion of the email address.
domain - domain name portion of the email address.
link - [optional] link text (defaults = email address).
To use this function, you first need an element in the page where the address will be placed and that element must have an id.
<span id="mail-link"></span>
Next the function needs to be called. The arguments to the function are the element id, the user name, the domain name, and the optional link text. As an example, if I wanted to send an email to glenn@gp_tools.com, I would call:
Gp.Basic.emailLink("mail-link", "glenn", "gp_tools.com");
Or if I wanted something besides the email address as the link text, I would add that as a fourth argument.
Gp.Basic.emailLink("mail-link", "glenn", "gp_tools.com", "Email me here.");

Gp.Basic.fixEmailLinks()
Looks for all elements in a page using the class "gp-send-mail" and for each element found, inserts an email link into the element using custom data attributes specified on the element itself. Like the previous function, the email is inserted in such a way that makes it difficult for spam bots to find.

Note that this function is automatically called on document load meaning there will never be a reason to call it directly. To take advantage, you need to specify an element (usually a SPAN element) in your document for each email link in the page. Each of those elements must use the class "gp-send-mail". At a minimum, each element must contain a custom attribute for the user name and another custom attribute for the domain name. You can also specify an optional third custom attribute containing the link text. If you leave that last attribute off, the email address will be used.

So let's say you want put in an email link just like in the last function, you would need only the following HTML.
<span class="gp-send-mail" data-user="glenn" data-domain="gp_tools.com"></span>
Or if you wanted an additional link but wanted to specify link text other than the email address, you can add a custom attribute for that as in the following.
<span class="gp-send-mail" data-user="glenn" data-domain="gp_tools.com" data-link="Mail here!"></span>

Gp.Basic.insertCopyright()
Looks for an element with the id "gp-copyright" and if such an element is found, inserts a copyright message into that element based on the year of today's date.

Note that this function is automatically called on document load meaning there will never be a reason to call it directly. To take advantage, you need an element (such as a DIV or SPAN in your document with the id "gp-copyright". Additionally, you need to specify who to attribute the copyright to. This can be done through an additional attribute on the element "data-who". So if I want to attribute a copyright to myself, I would put the following in my page:
<span id="gp-copyright" data-who="Glenn Poorman"></span>
With that element in place, the rest is automatic. See the very bottom of this page to see what the output looks like.
Gp.Basic.openWindow(url, width, height, scrollbars)
Allows for another browser window to be opened using a specified url.
url - url of the file to open in the new browser window.
width - width of new browser window.
height - height of new browser window.
scrollbars - [optional] display scrollbars ("true" or "false").
This function can be called from another script or can be called right in the page. I've frequently used this function in the "onclick" handler for an HTML anchor. The link below was written:
<a href="#noop" onclick="Gp.Basic.openWindow('popup.html',500,600,true);">Click Me</a>

Gp.Basic.randomImage(id, howmany, location, suffix)
The random image function allows the website developer to display one out of a collection of images but have the specific image in the collection chosen at random every time the page is reloaded.
id - id of the <img> element that will display the random image.
howmany - number of images in the collection to choose from.
location - location of the images (optional, default = same as page).
suffix - [optional] suffix of the image file (default = "jpg").
In order for this to work, the images must all reside in the same location, must be of the same format (all of the file suffixes must match), and must follow a specific naming criteria. The images must be named using the image element id followed by a dash and a 1-based index. For example, if you have a collection of six images and the name you pass in is "picture", then it will be assumed that your collection of images looks like:
picture-1.jpg
picture-2.jpg
picture-3.jpg
picture-4.jpg
picture-5.jpg
picture-6.jpg
Next, is the image placement. This is done by simply putting an IMG element in your page and setting the element id to the common image name (minus any extra digits). Note that this is crucial. If the id and the name do not match, nothing will display. Going back to the six image collection shown above then, you would put the following in your page:
<img id="picture">
Lastly you need to call the function. This can either be done inline within a <script> tag or it can be done in the document ready function. I prefer the latter as it allows the rest of the page to load before downloading the image.
Gp.Basic.randomImage("picture", 6, ".", "jpg");
Note that since the last argument defaults to "jpg" and the files shown here are "jpg" files, this code could be written:
Gp.Basic.randomImage("picture", 6, ".");
Lastly, since the location parameter is also optional and defaults to the same location as the page, we could have written:
Gp.Basic.randomImage("picture", 6);

Gp.Basic.simpleImage(id, caption, thumbscale, float, description)

Requires LightBox

The simple image function is used to place a simple popup image into a web page. The single image file is displayed as a thumbnail by applying the scale parameter to the original width and height of the image. A link is also created so that the image will display at its default size when clicked. This function uses "LightBox" to display the image and it is assumed that the lightbox JS and CSS files are loaded.
id - id of the element to contain the image and the url of the image file.
caption - caption to display under the thumbnail image.
thumbscale - scale applied to image width and height for thumbnail display.
float - [optional] CSS "float" property to set on the element (default = "none").
description - [optional] text that will appear in the window when the image is selected.
Placement is done by putting a DIV element in your page and setting the id of the element to the relative image url. The file suffix on the element id is optional. If no suffix is given, ".jpg" is assumed.

The picture on the upper right (for example), is "picture-1.jpg" from the collection above. The element for that picture could be written in one of two ways:
<div id="picture-1.jpg"></div>
Since the default file suffix is "jpg", the element could also be written:
<div id="picture-1"></div>
Note that when the function call is made, the ids must match exactly.

Next, the function must be called for each image in your page. For the image we just talked about then, the function call would look like:
Gp.Basic.simpleImage("picture-1.jpg", "Simple Image", 0.4, "right", "Some dandelions");
Or if you left the file suffix off:
Gp.Basic.simpleImage("picture-1", "Simple Image", 0.4, "right", "Some dandelions");
The arguments are the id/url followed by a caption that will be displayed just under the image. Note that as in the element placement, the file suffix is optional but it is important to note that the id in the function call and the id in the DIV element must match exactly. The next argument is the scale that is applied to the image width and height in order to display the thumbnail. Last is the float property. This can have any value that the CSS "float" property can have. The argument can also be left off in which case the default of "none" will be used.

CSS

The simpleImage function will automatically add the class gp-simple-image to the containing element allowing control over all manner of formatting options. The default style is shown below:
.gp-simple-image { text-align : center; }
